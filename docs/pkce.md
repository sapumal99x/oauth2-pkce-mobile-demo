# PKCE in this Demo

## What is PKCE?

PKCE (Proof Key for Code Exchange) is an OAuth2 extension that protects the authorization code flow from code interception attacks.

## Why mobile apps need it

Mobile apps are public clients (they cannot safely store a client secret). PKCE adds a one-time proof so the code alone is not enough to get a token.

## code_verifier vs code_challenge

- `code_verifier`: a high-entropy random string generated by the mobile app
- `code_challenge`: `BASE64URL(SHA256(code_verifier))` sent in the authorize request

During token exchange, the server hashes the received verifier and compares it with the original challenge.

## SHA256 + Base64 URL encoding

1. Compute SHA256 hash of the verifier.
2. Encode hash as base64.
3. Convert to base64url format:
   - `+` -> `-`
   - `/` -> `_`
   - remove trailing `=`

## Where PKCE is used in this project

- Mobile:
  - `apps/mobile-app/src/pkce.ts`: generate verifier/challenge
  - `apps/mobile-app/App.tsx`: send challenge, then send verifier at `/api/token`
- Server:
  - `apps/auth-server/src/app/api/authorize/route.ts`: stores challenge with auth code
  - `apps/auth-server/src/app/api/token/route.ts`: recomputes challenge and validates match
