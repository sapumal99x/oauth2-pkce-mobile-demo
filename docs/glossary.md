# Glossary

- **OAuth2**: Authorization framework that allows delegated access to protected resources.
- **PKCE**: Proof Key for Code Exchange, an extension that protects authorization code flow for public clients.
- **Authorization Code**: Short-lived code returned after user authentication, exchanged later for a token.
- **Access Token**: Credential used by client applications to access APIs.
- **Redirect URI**: Callback URI where the authorization server sends the code.
- **Deep Linking**: Mechanism for opening a specific screen in a mobile app via URI scheme.
- **SSO**: Single Sign-On, one login session used across multiple applications.
- **Identity Provider (IdP)**: Service that authenticates users and provides identity data (for example, Google).
